## Product and Strategy Work Rules

When creating PRDs, product specs, executive briefs, roadmaps, or operating models:

- Start with the problem being solved.
- Make the customer/user/operator pain explicit.
- Define the mechanism, not just the vision.
- Include success metrics.
- Separate discovery, design, build, launch, adoption, and measurement.
- Include risks, assumptions, tradeoffs, and governance where relevant.
- Make the output usable by founders, executives, software architects, and product teams.
- Prefer tables only when they improve decision-making.
- If creating a roadmap, include sequencing logic and why something should be built first.
- If discussing AI agents, define boundaries, tool access, audit logs, human review, and deterministic vs. interpretive behavior.

## Important Project Context

### Compassion Benchmark

Compassion Benchmark is intended to become a credible, evidence-based research and daily briefing destination for institutional compassion. It should feel more like a serious intelligence product than a feel-good nonprofit site.

Key dimensions:

- Awareness
- Empathy
- Action
- Equity

The updates page should be compelling enough that people want to visit every day for evidence-based insights about governments, corporations, AI labs, robotics companies, institutions, and global events.

### Ledgerium AI

Ledgerium AI is a Continuous Improvement Operating System for AI-native organizations. It should connect workflow recording, deterministic process derivation, SOPs, metrics, Kaizen, DMAIC, BAM scheduling, and ROI validation.

Core principle: turn real work behavior into measurable, improvable, auditable operating systems.

### Mediafier / Agent-First AI Media Platform

This work centers on semantic capabilities, governed MCP servers, media operations, and API/tool capability mapping. Claude should treat semantic capabilities as the bridge between user intent and executable tools.

Important concepts:

- Semantic Capability Platform
- Capability compiler
- OpenAPI-to-MCP generation
- Credential plane
- Tool call audit schema
- Append-only auditability
- Composite and atomic media workflows
- Enterprise governance from the start

### TraceWorks / Workflow Recorder

TraceWorks is a privacy-first universal workflow recorder concept that captures structured interaction events into immutable JSON, deterministically derives steps, activities, process maps, SOPs, and metrics, and supports non-authoritative interpretive AI insights.

Core principles:

- Raw data immutability
- Deterministic derivation pipelines
- Human-inserted session markers
- Optional AI interpretation
- Full traceability
