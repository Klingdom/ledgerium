# build-loop.md

Purpose:
Turn prioritized backlog items into working product increments.

Inputs:
- system.md
- memory.md
- backlog.md
- metrics.md
- decisions.md

Process:
1. Read current context and constraints
2. Select the top unfinished backlog item
3. Break it into implementation steps
4. Assign tasks to the appropriate specialist roles
5. Implement changes
6. Run validation checks
7. Update memory.md if system understanding changed
8. Update experiments.md if a hypothesis was tested
9. Mark backlog progress
10. Commit changes with a clear message

Exit Criteria:
- A working increment exists
- Validation completed
- Relevant docs updated
- Progress committed
