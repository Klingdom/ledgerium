# improve-loop.md

Purpose:
Continuously improve architecture, UX, performance, maintainability, and clarity.

Inputs:
- current repo state
- metrics.md
- memory.md
- experiments.md

Process:
1. Evaluate current repo against metrics
2. Identify the single highest-impact improvement
3. State the hypothesis
4. Implement improvement
5. Measure or assess impact
6. Record result in experiments.md
7. Update memory.md with durable learning
8. Commit changes

Exit Criteria:
- One measurable or clearly justified improvement completed
- Learning recorded
- Repo integrity maintained
