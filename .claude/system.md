# System

This folder stores the operating system for Claude-based execution.

## Working rules
- Always anchor work to a backlog item, active task, defect, or explicit opportunity.
- Prefer small, validated progress over large unvalidated changes.
- Keep a clean chain of reasoning across backlog, decision, experiment, and metric.
- When uncertainty is high, design an experiment instead of pretending certainty.

## File usage
- `backlog.md`: prioritized work and opportunities
- `memory.md`: durable context, current state, and notable learnings
- `decisions.md`: decision log with rationale and consequences
- `experiments.md`: tests of hypotheses, with outcomes
- `metrics.md`: KPIs and operational indicators

## Handoff rule
Any substantial work should leave behind enough context for another agent to continue immediately.
