# Competitive / Market-Attraction Assessment
**Ledgerium AI** · 2026-06-14 · Analysis only (read-only agent output, persisted by coordinator)

## Landscape (mid-2026)
- **Scribe / Scribe Optimize** — THE consequential threat. $75M Series C, $1.3B valuation (Nov 2025);
  Optimize captures real browser workflows → maps + bottlenecks + automation + ROI; 10M+ workflows,
  40k+ apps, 5M users, Fortune-500 penetration; 2026 moves: Magic Edit, MCP server, doc import. Core
  output stays screenshot-annotated guides. Weakness: intelligence layer is newer/less mature than its SOP layer.
- **KYP.ai** — agentic process intelligence; desktop+web capture, BPMN via genAI, ROI-ranked automation,
  generates agent code (UiPath/SAP). Strong on agent-code + BPMN.
- **Celonis / UiPath / SAP Signavio** — enterprise process mining (Celonis = 2026 Gartner PI Leader);
  require ERP event logs + months-long deployments; NOT Ledgerium's ICP (CIO/COO, $250k+).
- **Tango / Glitter / Guidde / Floik / iorad / MS Process Advisor** — lightweight screenshot/guide tools;
  table-stakes capture, no analytics depth.

## Ledgerium's differentiated position
Genuine white space at the ICP (ops / process-improvement, SMB–mid-market) between screenshot-SOP tools
(easy capture, zero analytics) and process-mining (deep analytics, zero capture ease, ERP-tied). Scribe
Optimize is the only credible attempt to close it bottom-up — but its intelligence is 6–12 months old.
**Defensible moat:** (1) per-figure **evidence anchoring**; (2) **determinism/reproducibility** (table-stakes
for Six Sigma/ISO; no doc tool has it); (3) **observed-only, never inferred**; (4) auto **Visio-grade map +
honest multi-run variant analysis** from a few real recordings. **Table-stakes (not differentiating):**
Chrome capture, AI step text, SOP output, PDF export.

## Attraction gaps vs competitors
1. **No screenshots in SOPs/Reports (HIGH)** — the single most visible gap vs Scribe/Tango; our report is
   analytically superior but visually abstract.
2. **No BPMN export (MED-HIGH)** — blocks formal BPM / ISO / lean buyers; KYP/Celonis/UiPath/Signavio have it.
3. **Browser-only capture (MED)** — no desktop/thick-client (SAP GUI, Workday client); Scribe/Celonis/KYP do.
4. **No integrations (MED)** — Scribe pushes to Confluence/Notion/Slack/700+; our output is self-contained.
5. **No social proof / case studies (HIGH for conversion)** — zero named users/quotes; ops buyers need peer proof.
6. **Org-scale multi-participant discovery (LOW-MED, rising)** — we operate at recording level vs Scribe's org-wide.

## Highest-impact attraction moves
1. **Screenshots per recorded step** in SOP + Report (extension already tracks events; additive) — closes the
   #1 table-stakes gap.
2. **3–5 named case studies** with before→recording→finding→action→outcome — unlocks self-serve + peer conversion.
3. **BPMN 2.0 export** from the existing map — opens the formal-BPM buyer segment Scribe doesn't target.
4. **Send-to Confluence/Notion/Slack** — removes the last value-delivery friction; drives sticky adoption.
5. **"Record + compare" team mode** — position multi-run variant analysis as consent-based team discovery
   (more defensible than Scribe's passive monitoring).

## Grade: B−
Strong, genuinely-differentiated position in real white space, with analytical depth ahead of Scribe's
intelligence layer — held back by table-stakes visual gaps (screenshots) and zero social proof.
**Biggest risk:** Scribe Optimize closing the white space within 18–24 months with its distribution +
funding. Window: establish named-customer proof + close screenshots + move into BPMN/formal-BPM before
Optimize matures.
