## 1. What Is Ledgerium AI?

Ledgerium AI is a browser extension that records your workflow activity and transforms it into structured, auditable process intelligence. Unlike screen recorders or AI summarizers, Ledgerium captures the **evidence** of what you actually did — every click, every form entry, every navigation — and deterministically converts that evidence into:

- **Workflow Steps** — meaningful actions grouped by intent
- **Process Maps** — visual flowcharts of your workflow
- **Standard Operating Procedures (SOPs)** — step-by-step instructions
- **Workflow Reports** — comprehensive PDF-ready reports

**Core principle**: Reality before opinion. Evidence before interpretation. Every output traces back to source events.

### What It Is NOT

- Not a screen recorder (no video or screenshots)
- Not a surveillance tool (you control when recording starts and stops)
- Not an AI summarizer (outputs are deterministic, not generated by a language model)
- Not a keylogger (it captures actions and targets, never passwords or sensitive values)

## 2. Installation

### From a ZIP File (Developer/Tester Mode)

1. **Download** the `ledgerium-extension.zip` file
2. **Unzip** it to a permanent folder on your computer
   - Example: `C:\Users\YourName\Extensions\ledgerium-extension\`
   - **Important**: Don't unzip to a temporary folder — Chrome needs the folder to persist
3. Open **Google Chrome**
4. Navigate to `chrome://extensions/`
5. Toggle **Developer mode** ON (switch in the top-right corner)
6. Click **Load unpacked**
7. Select the unzipped folder (the one containing `manifest.json`)
8. The Ledgerium AI icon appears in your browser toolbar

### Verifying Installation

- Look for the **Ledgerium AI** icon in your Chrome toolbar (top-right)
- If you don't see it, click the puzzle piece icon (Extensions menu) and pin Ledgerium AI
- Click the icon — a sidebar should open on the right side of your browser

### Updating the Extension

If you receive a new version:

1. Unzip the new file to the **same folder** (overwrite existing files)
2. Go to `chrome://extensions/`
3. Find Ledgerium AI Recorder
4. Click the **reload** button (↻ circular arrow)
5. Close and reopen the sidebar

Click the **Ledgerium AI** icon in your toolbar. The sidebar opens on the right side of your browser window.

## 4. Recording a Workflow

### Starting a Recording

1. Enter an activity name in the text field
2. Click **Start recording**
3. The sidebar shows "Initializing session…" briefly
4. The header changes to show a red **Recording** badge and a timer
5. A red pulsing dot appears in the header — recording is active

### What Happens During Recording

Once recording starts:

- **Every tab** you interact with is being captured
- Clicks, form inputs, navigations, file uploads — all recorded
- Events appear in real-time in the sidebar as **workflow steps**
- The timer in the header shows elapsed time
- The event counter shows how many raw events have been captured

### What Is Captured

| Action | What Ledgerium Records |
|---|---|
| Clicking a button | Target element, label, selector |
| Typing in a form field | Field name, that input occurred (NOT the value) |
| Navigating to a page | URL, page title, route |
| Switching tabs | Domain change, new application context |
| Submitting a form | Form target, submission event |
| Uploading a file | File input interaction |
| Using keyboard shortcuts | Enter, Escape, Tab key intents |
| Modal dialogs appearing | Modal open/close events |
| Error messages displaying | Error detection (visual signals) |

### What Is NOT Captured

| Not Captured | Why |
|---|---|
| Passwords | Automatically detected and excluded |
| Text you type | Only field names are recorded, not values |
| Screenshots or video | Event-based, not visual recording |
| Mouse movements or scrolling | Filtered as noise |
| Hovering over elements | Not meaningful for process documentation |
| Activity in other browsers | Chrome extension, Chrome only |

## 5. During Recording — The Live Step Feed

While recording, the sidebar shows a **real-time step feed** — a scrolling list of workflow steps that builds as you work.

### Understanding Step Cards

Each step card shows:

- **Icon** — indicates the step type:
  - **📋** Form submission (fill and submit)
  - **→** Navigation (click leading to a page change)
  - **↺** Repeated action (rapid clicks deduped)
  - **·** Single action (click, input, etc.)

- **Title** — a human-readable description of what happened:
  - "Navigate to Gmail Inbox"
  - "Enter Subject"
  - "Click Send ‪(Ctrl-Enter)‬"
  - "Attach file"
  - "Enter A18" (spreadsheet cell)

- **Page Label** — which application/page this happened on (e.g., "Mail", "Docs")

- **Confidence Pips** — 0 to 3 filled circles indicating how confident the system is about this step grouping:
  - ●●● = High confidence (form submission, navigation)
  - ●●○ = Medium confidence (data entry, single action with label)
  - ●○○ = Lower confidence (generic click without clear label)

- **Event Count** — how many raw events make up this step

### Provisional vs. Finalized Steps

- **Finalized steps** appear in full color — the system has determined the step boundary
- **Provisional steps** appear slightly faded with "…" — the system is still accumulating events and may adjust this step

Provisional steps become finalized when a boundary is detected (navigation, target change, time gap, etc.).

### Auto-Scrolling

The step feed automatically scrolls to show the latest step. You can scroll up to review earlier steps without interrupting the recording.

## 6. Pausing and Resuming

### Pausing a Recording

During an active recording:

1. Click **Pause** in the control bar
2. The header changes to show a yellow **Paused** badge
3. The step feed freezes (slightly faded)
4. The timer stops counting
5. No events are captured while paused

### Why Pause?

- Taking a phone call during a workflow
- Looking up information that isn't part of the process
- Stepping away temporarily
- Avoiding capturing irrelevant activity

### Resuming

1. Click **Resume** in the control bar
2. The header returns to red **Recording** badge
3. The timer resumes
4. Events start being captured again
5. The pause interval is recorded in session metadata

### Pause Intervals

Pause intervals are tracked in the session metadata. The workflow report distinguishes between active time and paused time, so your metrics reflect actual working time.

## 7. Stopping a Recording

### How to Stop

1. Click **Stop & review** in the control bar
2. The sidebar shows "Processing session" with a spinner
3. The system:
   - Finalizes all pending steps
   - Builds the complete session bundle
   - Generates the workflow report
   - Saves to activity history
   - Transitions to the review screen

### What Happens Behind the Scenes

When you press Stop:

1. All content scripts in all tabs stop capturing
2. Live steps are finalized (provisional steps become final)
3. Events are segmented into deterministic workflow steps
4. SHA-256 hashes are computed for data integrity
5. A canonical `workflow_report.json` is generated
6. The session bundle is persisted to browser storage
7. If an upload URL is configured, the bundle is uploaded

This process typically takes 1-3 seconds.

## 8. Reviewing Results

After stopping, the sidebar shows the **review screen** with three tabs: **Map**, **SOP**, and **Export**.

### Session Header

At the top of the review screen, you'll see:

- Green dot + "SESSION COMPLETE" label
- Your activity name
- Step count and event count

### 8.1 Map Tab

The Map tab shows an interactive **process map** — a visual flowchart of your workflow.

**What you see:**

- **Start node** (top) — beginning of the workflow
- **Step nodes** — each meaningful action in your workflow, connected by arrows
- **End node** (bottom) — end of the workflow
- **Transition labels** — on the arrows between steps (e.g., "Page navigation", "Form submitted")

**Node colors** indicate step type:

| Color | Step Type |
|---|---|
| Teal | Navigation |
| Blue | Form submission |
| Orange | Repeated action |
| Slate/Gray | Single action |
| Red | Error handling |
| Purple | Annotation |

**Interacting with the map:**

- Click any step node to open the **Step Drawer** with detailed information
- The drawer shows:
  - Step title and category
  - Operational definition (what this step does)
  - Purpose in the workflow
  - Systems and domains involved
  - Inputs and outputs
  - Completion condition
  - Duration and confidence
- Click the close button (×) or outside the drawer to return to the map

### 8.2 SOP Tab

The SOP (Standard Operating Procedure) tab shows a **human-readable, step-by-step procedure** for the recorded workflow.

**Sections:**

1. **Title and Purpose** — SOP title (e.g., "SOP: Email Customer AI Report") and a brief purpose statement

2. **Systems** — badges showing which applications/tools were used (e.g., "Mail", "Docs")

3. **Estimated Time** — approximate duration based on the recording

4. **Steps** — numbered list of each step with:
   - Step number (colored by category)
   - Step title (e.g., "Enter To recipients")
   - Duration
   - Action description (what to do)
   - Warnings (if any — shown in amber)
   - **Evidence Events** (expandable) — click the "▶ N evidence events" label to see the raw events that produced this step:
     - Color-coded dots: teal (user action), orange (system event), purple (recorder event)
     - Event type labels
     - Target descriptions
     - [redacted] indicators for sensitive fields

5. **Notes** — metadata about how the SOP was generated

### 8.3 Export Tab

The Export tab provides **download and export options** for the recorded session.

**Session Statistics:**

| Metric | Description |
|---|---|
| Activity | Your activity name |
| Steps | Number of workflow steps derived |
| Events | Total events captured (user + system) |
| Duration | Total session time |
| Status | "complete" or "partial" |

**Systems Accessed:**

Badges showing each application used during the recording.

**Action Buttons (top to bottom):**

1. **📄 Download Workflow Report** — Opens a formatted HTML report in a new tab. Use Ctrl+P / Cmd+P to save as PDF. (See [Section 9](#9-downloading-the-workflow-report))

2. **Open Full Process Map ↗** — Opens the interactive process map in a full browser tab (larger view for complex workflows)

3. **Export Enriched JSON (with events)** — Downloads a JSON file containing all steps with their SOP descriptions AND the raw source events. Best for developers and integrations.

4. **Export Raw Session JSON** — Downloads the raw session bundle as JSON. Contains session metadata, all normalized events, derived steps, policy log, and manifest with hashes.

**Bottom Controls:**

- **Done** button — closes the review screen and returns to the home screen
- **Discard** link — deletes the session entirely (with confirmation)

## 9. Downloading the Workflow Report

The **Download Workflow Report** button is the primary way to get a professional, shareable document of your recorded workflow.

### How It Works

1. Click **📄 Download Workflow Report** on the Export tab
2. The button shows "Generating Report…" briefly
3. A new browser tab opens with a formatted HTML report
4. To save as PDF: press **Ctrl+P** (Windows/Linux) or **Cmd+P** (Mac)
5. In the print dialog, select **Save as PDF** as the destination
6. Click **Save**

### What the Report Contains

The workflow report includes six sections:

#### A. Header
- Report ID
- Activity name
- Session duration
- Generation timestamp

#### B. Executive Summary
- Total steps, phases, and tools used
- Applications accessed (as badges)
- Workflow confidence score
- Key observations (multi-tab workflows, send actions, file interactions, idle time)

#### C. Workflow Phases
- Grouped by application (e.g., "Mail Workflow", "Docs Workflow")
- Step count per phase

#### D. Workflow Steps Table
- Numbered list with columns: Step, Application, Duration, Events
- Every step from your workflow in order

#### E. Metrics Dashboard
- Total duration, navigations, clicks
- Inputs, file actions, send actions
- Visual stat cards

#### F. Standard Operating Procedure
- Phase-by-phase instructions
- Each step as an actionable instruction
- Completion criteria

#### G. Footer
- Schema version
- Recorder version
- Segmentation rule version
- "Generated by Ledgerium AI" attribution

### Tips for PDF Export

- Use **Ctrl+P** / **Cmd+P** in the report tab
- Select **Save as PDF** (not your printer)
- Set margins to **Default** or **Minimum** for best layout
- Enable **Background graphics** if you want the stat card backgrounds
- The report is designed for **US Letter** or **A4** paper size

## 10. Exporting Data

### Export Formats

| Format | Best For | Contains |
|---|---|---|
| **Workflow Report** (PDF) | Sharing with stakeholders, documentation | Formatted report with all sections |
| **Enriched JSON** | Developer integration, detailed analysis | Steps + SOP data + source events |
| **Raw Session JSON** | Data processing, debugging, archival | Complete bundle with hashes |
| **Full Process Map** | Presentations, team review | Interactive visual flowchart |

### Enriched JSON Structure

The enriched JSON contains each step with:
- Step definition (category, title, confidence)
- SOP step (action, instructions, expected outcome)
- Source events (the raw evidence that produced this step)

### Raw Session JSON Structure

The raw bundle contains:
- `sessionJson` — session metadata (activity name, timestamps, pause intervals)
- `normalizedEvents` — all canonical events with page context and target summaries
- `derivedSteps` — deterministic step groupings with source event IDs
- `policyLog` — policy enforcement entries (blocked domains, redacted fields)
- `manifest` — version numbers and SHA-256 file hashes for integrity verification

## 12. Discarding a Recording

### During Recording

Click **Discard** in the control bar. This immediately:
- Stops all event capture
- Clears all recorded data
- Returns to the home screen
- The recording is NOT saved to history

### After Recording (Review Screen)

Click **Discard** on the Export tab. This:
- Removes the current session from the review screen
- Returns to the home screen
- Note: The session is ALREADY saved to history when recording stops. Discarding from the review screen just closes the review — the history entry remains.

### No Undo

Discarding during recording is permanent. The events captured so far are lost. If you're unsure, use **Pause** instead.

## 13. Multi-Tab Workflows

Ledgerium AI excels at recording workflows that span multiple browser tabs and applications.

### How It Works

When you start recording:
- **Every open tab** with a loaded page begins capturing events
- **New tabs** opened during recording are automatically captured
- **Tab switches** are detected as workflow transitions
- Events from different tabs are merged into a single timeline

### Example: Email + Spreadsheet Workflow

1. Start recording: "Email Customer AI Report"
2. Switch to Gmail tab → Ledgerium captures Gmail interactions
3. Compose email → Steps: "Enter To recipients", "Enter Subject", "Write Message Body"
4. Attach a file → Step: "Attach file"
5. Send email → Step: "Send ‪(Ctrl-Enter)‬"
6. Switch to Google Sheets tab → Domain change detected → New phase begins
7. Enter data in cells → Steps: "Enter A18", "Enter B18", "Enter C18"
8. Stop recording → Complete workflow with 2 phases

### How Steps Are Separated Across Tabs

The segmentation engine detects step boundaries when:
- You switch to a tab on a **different domain** (e.g., Gmail → Sheets)
- The **route changes** within the same domain (e.g., Inbox → Compose)
- You interact with a **different UI element** after a brief pause
- You click an **action button** (Send, Save, Submit)
- More than **45 seconds** pass between actions (idle gap)
- You **submit a form**

### Tips for Multi-Tab Recording

- Keep irrelevant tabs closed to reduce noise
- Perform the workflow naturally — Ledgerium adapts to your pace
- If you need to reference something unrelated, use **Pause** first
- The report will show separate phases for each application

## 14. Privacy and Security

### What Ledgerium Protects

| Protection | How |
|---|---|
| **Passwords** | Automatically detected (input type="password") and excluded entirely |
| **Sensitive fields** | Fields matching patterns like "secret", "token", "SSN", "credit card" are redacted |
| **Form values** | The content you type is NOT captured — only field names and that input occurred |
| **Query parameters** | URLs are stripped of query strings and fragments to prevent PII leakage |
| **Email addresses** | Labels that look like email addresses are filtered out |

### Redaction Transparency

When a sensitive field is detected, Ledgerium:
1. Emits a `system.redaction_applied` transparency event
2. Records that redaction occurred in the policy log
3. Shows `[redacted]` in the SOP evidence events

### Domain Filtering

If configured, Ledgerium can:
- **Block specific domains** — events from blocked domains are discarded with a `system.capture_blocked` event
- **Allow only specific domains** — only events from listed domains are captured

### Where Data Is Stored

- **During recording**: Events are held in the browser's service worker memory
- **After recording**: The session bundle is saved to Chrome's local storage (on your machine)
- **Upload (if configured)**: The bundle can be uploaded to a configured server endpoint
- **No cloud storage by default**: Your data stays on your machine unless you configure upload

### Data You Control

- **Start/Stop**: You decide when recording begins and ends
- **Discard**: You can delete any recording at any time
- **Export**: You control what data leaves your browser
- **History**: Old recordings are automatically pruned (max 25)

## 15. Troubleshooting

### "The sidebar won't open"

- Check that Developer mode is enabled at `chrome://extensions/`
- Verify the extension is loaded and enabled (toggle switch ON)
- Try clicking the Ledgerium icon again
- Reload the extension: go to `chrome://extensions/`, find Ledgerium, click the reload (↻) button

### "Recording started but no events appear"

- Make sure you're interacting with tabs (clicking, typing, navigating)
- Check that the tab has finished loading (wait for the page to fully render)
- Some pages with heavy iframe usage (like Gmail) may take a moment to start capturing
- The "Waiting for activity…" message is normal — events appear after your first interaction

### "Steps aren't splitting correctly"

- The segmentation engine uses deterministic rules:
  - Domain changes always split steps
  - Route changes within the same domain split steps
  - 2+ second gaps between different targets split steps
  - Action buttons (Send, Save, Submit) trigger step boundaries
- If steps seem too coarse, try pausing between distinct actions
- If steps seem too granular, interact faster without pauses between related actions

### "The process map screen says 'Generating process intelligence…'"

- Wait a few seconds — the engine may be processing
- If it persists, check the browser console for errors (right-click sidebar → Inspect → Console)
- Try stopping and starting a new recording
- If the issue persists, reload the extension at `chrome://extensions/`

### "The Download Workflow Report button shows 'Report Generation Failed'"

- The button will auto-reset after 3 seconds — try clicking again
- If it keeps failing, switch to the Export tab and use "Export Enriched JSON" as an alternative
- Check the browser console for specific error messages

### "My history is empty after restarting Chrome"

- History is stored in Chrome's local storage — it should persist across restarts
- If the extension was reloaded or reinstalled, history may be lost
- Make sure you're not running in Incognito mode (storage doesn't persist)

### "The extension icon disappeared"

- Click the puzzle piece icon (Extensions menu) in the Chrome toolbar
- Find Ledgerium AI Recorder
- Click the pin icon to keep it visible

### "Events from a specific site aren't being captured"

- Verify the site isn't in a blocked domains list (if configured)
- Some browser-internal pages (`chrome://`, `about:`, extension pages) cannot be captured
- Pages with strict Content Security Policies may block the content script

## 17. Frequently Asked Questions

### General

**Q: Does Ledgerium record my screen?**
A: No. Ledgerium captures structured events (clicks, inputs, navigations), not video or screenshots.

**Q: Can my employer see what I record?**
A: Only if you configure an upload URL. By default, all data stays on your local machine in Chrome's storage.

**Q: Does it work in Incognito mode?**
A: Chrome extensions can be enabled for Incognito, but storage won't persist between sessions.

**Q: Does it work with other browsers?**
A: Currently Chrome only (Manifest V3 Chrome extension).

### Recording

**Q: Can I record two workflows at the same time?**
A: No. One recording session at a time. Stop the current one before starting a new one.

**Q: What happens if Chrome crashes during recording?**
A: The session metadata is persisted. On restart, the extension attempts to resume, but events captured before the crash may be lost (events are held in memory until Stop is pressed).

**Q: Is there a time limit on recordings?**
A: No hard limit. The extension uses a keepalive alarm to prevent the service worker from sleeping. Very long recordings (hours) will accumulate more events and larger bundles.

**Q: Can I edit the steps after recording?**
A: Not currently. Steps are derived deterministically from events. A future version may support user annotations and corrections.

### Data

**Q: How long are recordings kept?**
A: Up to 25 recordings in Chrome local storage. Oldest are removed when the limit is reached.

**Q: Can I import a recording from someone else?**
A: Yes. Open the Full Process Map viewer (Open Full Process Map ↗ button), and drag-drop a JSON bundle file onto the upload area.

**Q: What's the difference between "Enriched JSON" and "Raw JSON"?**
A: Raw JSON is the session bundle as recorded. Enriched JSON adds SOP descriptions and the source events for each step — better for human review and integration.

### Reports

**Q: Why doesn't it generate a PDF automatically?**
A: Chrome extensions can't silently create PDFs. The report opens as HTML — use your browser's Print → Save as PDF for the highest quality output.

**Q: Can I customize the report template?**
A: Not yet in the current version. The report template is embedded in the extension. A future version may support custom templates.

**Q: What is "workflow confidence"?**
A: A weighted average of individual step confidence scores (0 to 1). Higher confidence means the engine is more certain about how events were grouped into meaningful steps.

## 18. Technical Details

### System Requirements

- Google Chrome version 116 or later (Manifest V3 support)
- Any operating system that runs Chrome (Windows, macOS, Linux, ChromeOS)
- No additional software, servers, or accounts required

### Extension Permissions

| Permission | Why It's Needed |
|---|---|
| `activeTab` | Read current tab information when you click the icon |
| `scripting` | Inject content scripts to capture page events |
| `storage` | Save recordings, history, and settings locally |
| `sidePanel` | Display the sidebar interface |
| `tabs` | Send messages to all tabs during recording |
| `alarms` | Keep the service worker alive during long recordings |
| `<all_urls>` | Content scripts must run on any page you visit |

### Data Integrity

Every session bundle includes a manifest with SHA-256 hashes of:
- `session.json` — session metadata
- `normalized_events.json` — all processed events
- `derived_steps.json` — workflow steps
- `policy_log.json` — policy enforcement log

These hashes allow verification that the data has not been tampered with after recording.

### Determinism Guarantee

Given the same input events, the Ledgerium engine will always produce the same outputs:
- Same steps with same titles
- Same confidence scores
- Same SOP instructions
- Same process map nodes and edges

This is enforced by:
- No randomness in step derivation (step IDs are ordinal-based, not UUID)
- No AI/LLM in the transformation pipeline
- Hardcoded, versioned segmentation rules
- Pure functions with no side effects
