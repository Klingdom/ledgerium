# Ledgerium AI — Help Documentation Source

This directory holds the **source-of-truth** markdown for Ledgerium's user-facing
help and training documentation. The rendered docs site at
[ledgerium.ai/docs](https://ledgerium.ai/docs) is generated by integrating these
files into `apps/web-app/src/app/(public)/docs/page.tsx` via the per-feature
`.integration.md` plans.

## Authoring

Don't write these files by hand — invoke the **`docs-engineer`** agent. It runs
the full pipeline:

1. Reads the feature registry at `apps/web-app/scripts/lib/feature-registry.ts`
2. Runs Playwright via `apps/web-app/scripts/capture-feature-screenshots.ts`
3. Produces `<feature-id>.md` (this directory)
4. Produces `<feature-id>.integration.md` (render plan for `frontend-engineer`)

Agent definition: `.claude/agents/docs-engineer.md`.

## Directory layout

```
docs/help/
├── README.md                              ← you are here
├── <feature-id>.md                        ← source-of-truth content
└── <feature-id>.integration.md            ← React patch instructions for docs/page.tsx
```

Screenshots referenced by the markdown live at:

```
apps/web-app/public/img/help/feature-<id>-<state>.png
```

…and are regenerated deterministically from the feature registry.

## Style guarantees

- Sentence case in headings ("Workflow detail view")
- No marketing adjectives (banned list in `.claude/agents/docs-engineer.md`)
- Verb-first imperatives in step lists
- Concrete numbers, not weasel words
- Audit-honest: if the UI says "Coming in an upcoming release", the doc says so verbatim
- Every viewState gets documented (empty, loaded, error, gated, etc.)

## Determinism

Same registry → same screenshots → same markdown → same rendered docs.

To re-capture all screenshots:

```bash
# Start dev server first (in another shell)
pnpm --filter @ledgerium/web-app dev

# Capture everything
pnpm --filter @ledgerium/web-app exec tsx scripts/capture-feature-screenshots.ts

# Or one feature
pnpm --filter @ledgerium/web-app exec tsx scripts/capture-feature-screenshots.ts --feature dashboard

# Or one state
pnpm --filter @ledgerium/web-app exec tsx scripts/capture-feature-screenshots.ts --feature dashboard --state empty
```

## Invoking the agent

From a coordinator session:

```
Agent({
  description: "Document dashboard feature",
  subagent_type: "docs-engineer",
  prompt: "Document the workflow library dashboard feature.
           Follow the DELIVERY RECIPE in .claude/agents/docs-engineer.md.
           Target: docs/help/dashboard.md + integration plan."
})
```

Or via the user shortcut: ask the coordinator to delegate to `docs-engineer`
and name the feature(s) you want documented.
