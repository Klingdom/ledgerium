# Demo Screenshots Manifest

Single source of truth for the marketing-site product-demo imagery. Updated when any demo screenshot is refreshed so future engineers know what each image is, where it came from, and how to regenerate it.

**Last refresh:** 2026-06-21 — all four core views recaptured from the **live app** (real-app Playwright capture, no mockups).

## Why this exists

The marketing demo previously rendered from **static HTML mockups** (`public/dashboard.html`, `public/samples/*.html` → `public/img/screenshot-*.png`). Those mockups had no link to the real app and silently drifted out of date (missing the portfolio timestudy band, DFG performance overlay, lead-first report, stat columns + N-attribution, etc.). They are now replaced with real current-state captures under `public/img/demo/`.

## Capture method

Real-app Playwright capture against the live dev server (`scripts/capture-feature-screenshots.ts` + `scripts/lib/feature-registry.ts`), authenticated, output at 900×560 viewport × 2 device-scale-factor (retina). Source captures land in `public/img/help/feature-*.png`; the marketing copies live in `public/img/demo/`.

### Reproduce

```bash
# from apps/web-app/
AUTH_EMAIL=demo@ledgerium.ai AUTH_PASSWORD=<demo-pw> \
  node_modules/.bin/tsx scripts/refresh-auth-state.ts      # writes e2e/.auth/user.json
node_modules/.bin/tsx scripts/capture-feature-screenshots.ts   # requires `pnpm dev` running

# then refresh the marketing copies:
cp public/img/help/feature-dashboard-loaded.png            public/img/demo/dashboard.png
cp public/img/help/feature-dashboard-interactive-column-picker.png public/img/demo/dashboard-columns.png

```

## Honesty rules (do not violate)

- Every visible metric must come from a real seeded recording. **No fabricated/benchmark numbers** in demo imagery or captions — this protects the "measured, not estimated" moat.
- Do not show features that are not shipped (e.g., "AI Agent Composition" was removed 2026-06-21 — not yet shipped).
- Recapture rather than hand-edit. The whole point is the demo reflects what the product actually looks like.
