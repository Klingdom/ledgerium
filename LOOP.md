# Loop Rules

## Purpose
Loops should continuously improve the product, codebase, and operating system without creating chaos.

## Default loop sequence
1. Read current priorities and recent memory
2. Select one bounded work item
3. Plan the change
4. Execute the change
5. Validate with evidence
6. Record decisions, experiments, and metrics
7. Propose the next best task

## Stop conditions
Stop a loop when:
- acceptance criteria are met
- confidence drops due to missing context
- risk becomes too high for autonomous action
- repeated failed attempts indicate a need for human review

## Safety rules
- Do not perform broad rewrites unless explicitly prioritized.
- Do not chain major risky changes in a single loop.
- Prefer multiple safe loops over one heroic loop.
